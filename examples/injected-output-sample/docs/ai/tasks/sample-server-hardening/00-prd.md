# PRD

## Goal

Turn the injected sample into a believable post-upgrade repository snapshot
instead of a raw copy of the generic large example.

## User Value

Readers should be able to compare:

- a business repository before harness injection
- the same style of repository after harness injection
- the extra task evidence a large-mode upgrade adds

## Success Criteria

- task identity is business-oriented
- the evidence chain mirrors `.ai/state.json`
- the sample stays intentionally small and reviewable
