# Design Before Code

## Purpose

Before non-trivial implementation, convert intent into spec / scope / plan.

## Use When

Feature work, bug fix with unclear root cause, cross-file changes, and C++ system changes.

## Inputs

`AGENTS.md`, `docs/ai/*`, `.ai/spec.md`, `.ai/scope.md`, and `.ai/implementation-plan.md`.

## Process

Clarify goal, non-goal, affected files, risk, and validation before code changes.

## Output

A clear spec / scope / plan that can drive a bounded implementation.

## Do Not

Do not modify code before the plan is clear.
Do not expand scope.
Do not bypass human gates.
