# Bootstrap Local Agent

Use this prompt after cloning `Auto_AICoding_Harness` into a machine that should use its workflow.

## Prompt

You are configuring yourself to use this repository as an AI coding harness source.

1. Inspect this repository first:
   - `README.md`
   - `docs/install-targets.md`
   - `global/AGENTS.md.template`
   - `skills/README.md`
2. Identify your active AI coding surface:
   - Codex
   - Claude Code
   - GitHub Copilot / VS Code
   - another local agent
3. Install or link only the files that your surface can actually consume.
4. Prefer explicit user-level or project-level install locations documented in `docs/install-targets.md`.
5. Do not silently overwrite existing global instructions or skills.
6. If a target file exists, show the diff or create a backup before replacing it.
7. After setup, report:
   - installed instruction files
   - installed or linked skills
   - files skipped
   - required restart or reload steps

## Expected Result

The local agent can find:

- a short global instruction entrypoint derived from `global/AGENTS.md.template`
- repository-owned skills from `skills/`
- project-level `AGENTS.md` files generated by this harness in target repositories

The local agent must not treat this prompt as permission to install third-party dependencies or fetch unreviewed skills.
