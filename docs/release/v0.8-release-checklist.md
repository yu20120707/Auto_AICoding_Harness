# v0.8 Release Checklist

## Baseline

- Version: v1.3-skill-creator-zh-readme
- Scope: self-use Codex-only AI Coding Harness
- Default profile: cpp-linux-backend-system

## Core Commands

- [ ] ai-init small
- [ ] ai-upgrade large
- [ ] ai-status
- [ ] ai-review spec / plan / diff / final
- [ ] ai-approve spec / plan / diff / final
- [ ] ai-reject spec / plan / diff / final
- [ ] ai-context-pack
- [ ] ai-handoff

## Core Guarantees

- [ ] safe write: CREATED / SKIPPED / OVERWRITTEN
- [ ] --force backs up before overwrite
- [ ] context-pack / handoff do not advance state
- [ ] no third-party runtime dependencies
- [ ] subagent / skills are enhancement layers, not hard dependencies
- [ ] templates are the only source for generated project files
- [ ] core workflow is separated from cpp-linux-backend-system profile

## Profile Guarantees

- [ ] generated docs/ai contains C++ system guidance
- [ ] generated scripts are safe placeholders
- [ ] generated .codex/agents contains optional large-mode role templates
- [ ] generated .agents/skills contains optional large-mode local skill templates
- [ ] generated .agents/skills includes subagent-to-skill routing guidance
- [ ] generated .agents/skills contains provenance and adaptation notes
- [ ] repository includes Chinese README.zh-CN.md
- [ ] no business source directories are created
- [ ] no system dependencies are installed

## Tests

- `tests/test_ai_init_small.py`: small init, idempotence, force backup, status, exit codes
- `tests/test_ai_upgrade_large.py`: large upgrade behavior and backup semantics
- `tests/test_ai_review_diff.py`: diff review preconditions and waiting-gate transition
- `tests/test_ai_review_spec_plan_final.py`: spec/plan/final review materials and waiting-gate transitions
- `tests/test_ai_approve_reject_diff.py`: diff approve/reject behavior
- `tests/test_ai_approve_reject_all_gates.py`: approve/reject coverage for spec/plan/diff/final
- `tests/test_ai_context_handoff.py`: context-pack and handoff generation without state advancement
- `tests/test_cpp_profile_templates.py`: profile docs generation and safe script placeholders
- `tests/test_current_capabilities.py`: manifest and capability drift checks
- `tests/test_e2e_workflow.py`: end-to-end workflow to `DONE`
- `tests/test_subagent_templates.py`: large-mode subagent role template generation and safe-write checks
- `tests/test_skill_templates.py`: large-mode local skills template generation and safe-write checks

## Explicitly Not Implemented

- subagent execution
- skills installation
- automatic third-party skill fetching
- multi-profile marketplace
- formal packaging / release CLI
